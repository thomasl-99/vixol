## 1. Introduction

VIXOL ("we," "us," "our," or "Company") operates the vixol.cloud website and the @vixol/vixol npm package (collectively, the "Service"). This Privacy Policy explains how we collect, use, disclose, and safeguard your information when you use our Service.

Please read this Privacy Policy carefully. **If you do not agree with our policies and practices, please do not use our Service.**

## 2. What Information We Collect

### 2.1 Information You Provide Directly

**Account Information:**
- Email address
- Username
- Password (hashed, never stored in plain text)
- Payment information (processed by Stripe, not stored by us)

**API Usage Information:**
- Your queries (the text you submit)
- API keys you provide (stored encrypted)
- Usage patterns (timestamps, frequency)

### 2.2 Information We Collect Automatically

**Log Data:**
- IP address
- Browser type
- Pages visited
- Time and date of access
- Referring URL

**Query Analytics:**
- Query complexity classification
- Provider selected (Llama, Groq, Claude, Opus)
- Response latency
- Token usage
- Cost per query

**Device Information:**
- Device type
- Operating system
- Unique device identifiers

### 2.3 What We Do NOT Collect

❌ We do NOT store or collect:
- The content of your LLM responses
- Your personal health information
- Financial information beyond what Stripe processes
- Biometric data
- Location data (unless you opt-in)

   - Process payments

3. **Analytics & Optimization**
   - Improve routing accuracy
   - Identify service issues

   - Optimize performance

4. **Security**
   - Detect fraudulent activity
   - Prevent abuse
   - Monitor rate limits
   - Audit access logs

5. **Legal Compliance**
   - Respond to legal requests
   - Comply with regulations
   - Maintain audit trails
   - Enforce Terms of Service

### 3.2 What We DO NOT Do

❌ We do NOT:
- Sell your data to third parties

## 5. Data Security

### 5.1 How We Protect Your Data

✅ **Encryption:**
- HTTPS/TLS for all data in transit
- AES-256 encryption for sensitive data at rest
- API keys stored encrypted

✅ **Access Controls:**
- Role-based access control
- Password hashing (bcrypt)
- Multi-factor authentication available

✅ **Monitoring:**
- Rate limiting to prevent abuse
- Security headers (X-Frame-Options, CSP, etc.)
- Audit logging of all API calls
- Regular security audits

✅ **Infrastructure:**
- Hetzner managed cloud (ISO 27001 certified)
- Firewall protection (UFW)
- Tailscale VPN for internal communication
- No public API key exposure

### 5.2 What You Should Do

⚠️ **Your responsibility:**
- Keep your API keys confidential
- Use strong passwords
- Don't share your account
- Report suspicious activity immediately
